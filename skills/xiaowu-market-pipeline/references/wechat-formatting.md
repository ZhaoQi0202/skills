# 公众号排版与 HTML 生成规范

## 目标

解决以下常见问题：
- 小标题没有“一、二、三”编号
- 板块之间没有稳定留白
- 原文链接在公众号草稿中不可点击
- 临时手写 HTML 导致版式每次都漂移

## 强制规则

当正文要进入微信公众号草稿时：

1. 不要临时把 Markdown 粗糙替换成 `<p>`。
2. 优先使用 `scripts/render_wechat_article.py` 生成 HTML。
3. 渲染后的正文中，禁止残留任何 Markdown 标题符号，例如 `# `、`## `、`### `。
4. `render_wechat_article.py` 中，一级标题匹配必须严格区分 `# ` 与 `## `，避免把二级标题误判成一级标题并把 `#` 字符渲染进正文。
5. 固定使用以下一级标题：
   - 一、结论
   - 二、指数与市场情绪
   - 三、今日主线
   - 四、盘面理解
   - 五、催化与后续观察
   - 六、明日重点观察
   - 七、小五一句话收口
6. 每个一级标题前后都要显式插入留白段，而不是只依赖 Markdown 空行。
7. 每个自然段尽量短，避免连续大段文字堆叠。
8. 新闻链接不要只保留裸文本 URL；应输出真正的 HTML 超链接。
9. 为兼容公众号渲染，链接建议同时包含：
   - 可点击标题链接：`<a href="...">原文标题</a>`
   - 备用完整 URL 文本：`http://...`

## 推荐输入格式

将正文源文件写成结构化 Markdown：

- 文档第一行：`# 总标题`
- 一级章节：`## 一、结论`
- 正文段落：普通文本
- 列表项：`- ...`
- 新闻链接行支持以下两种：
  - `原文链接：https://example.com/a`
  - `- 链接：https://example.com/a`

## 渲染策略

`render_wechat_article.py` 应按以下策略输出：

- 标题：独立居中或独立强标题段
- 一级章节：强样式标题段，前后留白
- 标题解析优先级必须先判断总标题 `# `，再单独判断章节标题 `## `，且二者的正则不能互相吞掉前缀井号
- 普通段落：`<p>` + 合理行高与 margin
- 列表项：转为项目符号段落，不依赖浏览器默认 `ul/li`
- 链接：
  - 从 URL 中提取并包裹为 `<a href="...">...</a>`
  - 同时在下一行或同段中保留完整 URL 作为兜底

## 经验规则

### 1. 标题层级

不要只写“**结论**”。
要写成“**一、结论**”这类可扫描标题。

另外，公众号最终 HTML 中不得出现 `# 结论`、`## 结论` 这类 Markdown 残留；发稿前应先抽查渲染结果头部几段，确认标题已被正确转换。

### 2. 留白

公众号草稿渲染对空行并不稳定。
应通过显式的 HTML margin/padding 留白，而不是只靠 Markdown 换行。

### 3. 链接可点击性

如果只输出：

```text
原文链接：http://example.com
```

在公众号草稿中不一定会变成可点击。

应优先输出：

```html
<p>原文链接：<a href="http://example.com">证券时报原文</a></p>
<p style="color:#666; font-size:13px;">备用地址：http://example.com</p>
```

### 4. 不要过度复杂排版

避免在正文里使用：
- 表格
- 多栏布局
- 复杂嵌套列表
- 花哨颜色块

公众号正文更适合：
- 稳定段落
- 明确标题
- 少量强调
- 可点击链接

## 推荐用法

```bash
python3 skills/xiaowu-market-pipeline/scripts/render_wechat_article.py \
  --input /path/to/article.md \
  --output /path/to/article.html
```

然后再把输出的 `article.html` 交给：

```bash
python3 skills/xiaowu-market-pipeline/scripts/wechat_mp_publish.py ...
```
